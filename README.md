# Windows Project Backup Script (Batch & PowerShell)

이 스크립트는 Windows 환경에서 특정 프로젝트 폴더의 파일들을 선별하여 복사하고, 날짜별로 ZIP 압축 백업을 수행하는 배치 파일입니다.

## 주요 기능
- **선택적 백업**: 지정된 파일 목록만 추출하여 백업하거나, 목록이 비어 있을 경우 폴더 전체를 백업합니다.
- **자동 디렉토리 관리**: 백업 저장 경로가 존재하지 않을 경우 자동으로 생성합니다.
- **동적 파일명 생성**: 원본 폴더 이름과 현재 시간을 조합하여 [폴더명]_[yyyyMMdd_HHmm].zip 형식으로 저장합니다.
- **경로 구조 유지**: 압축 파일 내부에 원본의 상위 폴더 구조를 포함하여 복구 시 혼선을 방지합니다.
- **환경 호환성**: Windows 10 및 Windows 11의 기본 기능(Robocopy, PowerShell)만을 사용하여 별도 설치 없이 동작합니다.

## 사용 방법

1. backup_script.bat 파일을 다운로드합니다.
2. 텍스트 편집기(메모장 등)로 파일을 엽니다.
3. 상단의 '1. 사용자 설정' 섹션을 수정합니다.
   - SRC_BASE: 백업 대상인 원본 프로젝트 폴더 경로
   - BACKUP_ROOT: 압축된 백업 파일이 저장될 대상 경로
   - FILE_LIST: 백업할 개별 파일들의 상대 경로 (전체 백업 시 비워둠)
4. 수정된 파일을 저장한 후 실행합니다.

## 설정 예시

### 1. 특정 파일들만 선별하여 백업할 경우
```batch
set FILE_LIST=(^
"src\main\java\com\App.java"^
"src\main\resources\config.xml"^
)
```

### 2. 프로젝트 폴더 전체를 백업할 경우
```batch
set FILE_LIST=(^
REM 목록을 비워두면 전체 백업 모드로 동작합니다.
)
```

## 기술적 주의 사항
- **구문 규칙**: FILE_LIST 작성 시 줄 끝의 ^ 기호 직전에 공백이 포함되지 않도록 주의하십시오.
- **주석 처리**: 배치 파일의 괄호 블록 내부에서 주석을 작성할 때는 문법 오류 방지를 위해 반드시 REM 키워드를 사용해야 합니다.
- **실행 권한**: 시스템 폴더나 네트워크 드라이브 접근 시 권한 문제가 발생할 경우 '관리자 권한으로 실행'을 권장합니다.
